# UX Principles

## The One Question Rule

Every screen should answer exactly one question:

> What should I do next?

If a screen has multiple competing answers, simplify it.

## First-Time Confidence

A person using the product for the first time should feel confident within the first minute.

They should understand:

- What this is
- What to do next
- What will happen

## Simplicity Is Respect

Simplicity is not the absence of depth.

Simplicity is depth made usable.

Remove what does not serve the person.

## Build Confidence Before Complexity

Advanced capability should never appear before basic trust is established.

A companion introduces itself once.

Then it quietly gets out of the way.

## Intention Over Features

Do not organize products around internal features.

Organize them around human intentions.

Good:

- Speak with someone
- Hear more clearly
- Remember this conversation
- Tell a story

Avoid:

- Translation mode
- Medical mode
- Legacy mode
- Memory module

## No Hidden Doors

Core actions must never be hidden behind undocumented gestures.

If the user needs it, it should be visible.

Power-user shortcuts may exist, but never as the only path.

## Large, Calm, Clear

Design for people who may be tired, distracted, nervous, older, hard of hearing, or unfamiliar with AI.

Use large text.

Use clear hierarchy.

Use calm motion.

Use fewer choices.

## Remove Before Adding

Before adding a feature, ask what can be removed.

The best design decision may be subtraction.
