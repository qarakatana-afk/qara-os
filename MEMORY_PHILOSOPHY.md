# Memory Philosophy

Memory is an act of care.

It is not a database strategy.

## What We Remember

We remember only what helps people feel more understood.

Examples:

- Names of people the user chooses to remember
- Preferred languages
- Communication preferences
- Important relationships
- Ongoing stories
- Places that matter
- Topics that recur naturally
- Unfinished conversations
- User-selected preferences

## What We Do Not Remember

We do not remember things only because they improve engagement.

We do not center:

- Visit counts
- Click counts
- Session streaks
- Time away
- Artificial usage milestones

## Consent

People should understand what is remembered and why.

Memory should be inspectable.

Memory should be editable.

Memory should be deletable.

## Relationship Memory Over Analytics

Analytics describe product usage.

Relationship memory supports human connection.

Qara products prioritize relationship memory.

## The Memory Test

Before storing anything, ask:

> Will remembering this help someone feel more understood?

If no, do not store it.
