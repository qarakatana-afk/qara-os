# QARA OS Change Control v0.1

## Purpose

This document governs how QARA OS changes are introduced without compromising governance integrity.
It defines change handling only; it does not create new authority roles.

---

# Change Categories

## Routine Execution (Non-Material)

Routine execution under an existing ACTIVE Kernel Decision Record, with no material change in purpose, scope, risk, served group, or expected human outcome.

Approval Required:
No

---

## Documentation

Clarifications that do not change authority interpretation, Kernel behavior, or production policy.

Approval Required:
Responsible steward for the affected artifact scope

---

## Governance

Changes affecting:

- Authority
- Kernel
- Decision behavior
- Production policy

Approval Required:
As required by controlling authority (QS hierarchy) and responsible human steward accountability

Kernel review required before implementation.

---

# Required Change Record

Every approved change shall include:

- Description
- Reason
- Authority affected
- Kernel Decision Record reference (and Review Record reference when escalation or review occurs)
- Date
- Approver

---

# Audit

Every production change must remain traceable.

Historical records are never deleted.

Corrections are appended.

History is preserved.
